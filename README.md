# 🚗 Smart Used Car Export ERP

> **영업 총괄의 수출 속도 극대화**와 **회계 팀장의 세무 및 법적 리스크 관리**를 동시에 해결하기 위한 스마트 중고차 수출 ERP

---

# 📌 프로젝트 소개

기능과 구현에 중점보다
영업팀은 빠른 선적을 원하고 회계팀은 세무 및 법적 리스크를 막아야 하는
서로 충돌하는 요구사항을 기술적으로 해결하는 것이 핵심이라고 생각했습니다.

따라서 대시보드의 형태를 최소화하고 차량의 현재 상태와 수출로직에 대한 단계가 메인에 오는 것이 
사용자가 편리하고 빠르게 학습할 수 있는 방법이라고 생각했습니다.
시장과 사용자 맞춤 erp의 형태가 무엇인지 고민을 했으며 기업 형태에 맞는 형식은 자동차 수출을 위한 과정임을 주목했고 이에 따른 구현을 하였습니다.
<img width="2560" height="1281" alt="image" src="https://github.com/user-attachments/assets/05fc3b9c-f173-48b3-82ec-5069a9bbf152" />

- 증빙 누락
- VIN 압류·도난 차량 선적 위험
- 부가세 손실
- 매입·매출 언밸런스
- 회계팀과 영업팀의 지속적인 충돌

에 따른 정보를 모아볼 수 있게 구성하여 최종 안정형태까지 이끌어내는 것이 목표입니다.

이번 프로젝트는

> **빠른 선적을 원하는 영업팀**
은 검증이 끝난 차량부터 적재가 가능하고
> **법적·세무 리스크를 방어해야 하는 회계팀** 은
검증이 끝나지 않은 차량에 대한 일괄적인 처리를 가능하도록 했습니다.

을 목표로 개발하였습니다.

# 🏗 시스템 구성

```text
차량 매입

↓

모바일 등록 (Mock)

↓

ERP 자동 등록

↓

리스크 자동 검사

↓

회계 승인

↓
```

실제 서비스에서는 구매 담당자가 모바일을 통해 차량 정보를 등록하면 ERP에 자동으로 등록되는 구조를 고려하였습니다.

이번 과제에서는 Mock 데이터를 활용하여 구현하였습니다.

# 🚨 리스크 관리

ERP에서 가장 중요하게 생각한 부분은

> **리스크를 얼마나 빠르게 발견하고 대응할 수 있는가**

였습니다.

단순히 Error를 발생시키는 것이 아니라

- Error Class
- Error Code
- Error Level

을 통해 캡슐화하여 관리하도록 설계하였습니다.

## Error Level

| Level   | 의미     | 예시                        | 처리 방식      |
| ------- | -------- | --------------------------- | -------------- |
| Level 1 | INFO     | 증빙 완료, 승인 완료        | 일반 알림      |
| Level 2 | WARNING  | 증빙 누락, 부가세 위험      | 회계 검토 요청 |
| Level 3 | CRITICAL | VIN 압류, 도난 의심, 역마진 | 즉시 대응      |

---

## Critical 처리

다음과 같은 상황은

- VIN 압류
- VIN 도난 의심
- 역마진 발생
- 선적 불가 차량

모두 **Critical** 로 분류됩니다.

향후 실제 서비스에서는

- Slack
- Discord
- Microsoft Teams

등 메신저와 연동하여 즉시 알림을 보낼 수 있도록 고려하였습니다.

```text

↓

VIN 대기

↓

선적 보류

↓
```

## 회계팀 직권 보류

회계팀은 언제든

**직권 선적 보류**

를 실행할 수 있도록 구현하였습니다.

직권 보류 시

- 차량은 승인대기 상태로 변경
- 영업팀은 임의 선적 불가

즉,

**회계팀의 통제권을 시스템에서 보장**하도록 설계하였습니다.

# 🤝 딜러 관리

차량뿐 아니라

거래하는 딜러의 정보도 중요한 자산이라고 판단하였습니다.

관리 항목

- 명함
- 연락처
- 거래 차량
- 판매 이력

이를 통해

- 문제 차량 비율
- 거래 성향
- 우수 딜러 여부

등을 쉽게 확인할 수 있도록 구현하였습니다.
딜러가 판매한 차량에 몇 건의 문제가 있는지 쉽게 알 수 있으며 현재 문제가 생긴 사항을
딜러에게 한번에 전달할 수 있는 장점과 딜러에 대한 정보를 쉽게 찾을 수 있습니다.

추후에는 모바일앱에서 딜러 정보를 올려 놓고 직원들이 클릭하여 자동차를 매입 할 시 
기존에 등록된 딜러를 클릭함으로써 데이터를 관리합니다.

향후 **차량 매입 전략**에도 활용할 수 있도록 고려하였습니다.

# 💡 프로젝트에서 중요하게 생각한 점

이번 프로젝트는 기능 구현을 중점에 두지 않았습니다.
내가 만약 소비자라면 살 것인가?
유투브나 광고에서 보는 복잡한 erp의 형태보다 딱 필요한 기능만 배우기 쉽고 기존 프로세스에
녹여내는 점이 가장 중요하다고 생각했으며 깔끔한 디자인과 한눈에도 알아볼 수 있는 단계별 형태에 집중했습니다.
복잡한 형태는 erp 구매자의 기업형태를 생각해봤을 때 기존 프로세스를 이용한다거나 반감을 사는 성장하는 오래된 기업일 것 같았습니다.
따라서 통계 및 차트 시각화를 넣지 않았으며 그 외 다양한 부가적인 기능을 고려하지 않았습니다.

또한 수출 적재 문제가 생기는점은 처음 매입할 때라는 점이 중요했고 딜러 관리를 넣은 이유입니다.
앱을 개발해서 현장직이 차를 구매할때 정보를 넣고 명함을 찍어놔서 딜러를 관리하면 좋을 것 같습니다.

프로세스가 관리자 손안에 있으며 컨트롤 할 수 있는 점이 중요하다고 생각했습니다.
