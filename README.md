# 🚗 Smart Used Car Export ERP

> **영업 총괄의 수출 속도 극대화**와 **회계 팀장의 세무 및 법적 리스크 관리**를 동시에 해결하기 위한 스마트 중고차 수출 ERP

---

# 📌 프로젝트 소개

기능과 구현에 중점보다
영업팀은 빠른 선적을 원하고 회계팀은 세무 및 법적 리스크를 막아야 하는
서로 충돌하는 요구사항을 기술적으로 해결하는 것이 핵심이라고 생각했습니다.

- 증빙 누락
- VIN 압류·도난 차량 선적 위험
- 부가세 손실
- 매입·매출 언밸런스
- 회계팀과 영업팀의 지속적인 충돌

이번 프로젝트는

> **빠른 선적을 원하는 영업팀**

과

> **법적·세무 리스크를 방어해야 하는 회계팀**

두 부서 모두를 만족시키는 ERP를 목표로 설계하였습니다.

---

# 🎯 핵심 목표

기존 ERP처럼

- 무조건 선적을 막는 시스템 ❌
- 무조건 선적을 허용하는 시스템 ❌

이 아닌

> **차량의 위험도를 자동으로 판단하여 선적 여부를 결정하는 시스템**

을 목표로 개발하였습니다.

---

# 🏗 시스템 구성

```text
차량 매입

↓

모바일 등록 (Mock)

↓

ERP 자동 등록

↓

리스크 자동 검사

↓

회계 승인

↓

선적 진행
```

실제 서비스에서는 구매 담당자가 모바일을 통해 차량 정보를 등록하면 ERP에 자동으로 등록되는 구조를 고려하였습니다.

이번 과제에서는 Mock 데이터를 활용하여 구현하였습니다.

---

# 🚨 리스크 관리

ERP에서 가장 중요하게 생각한 부분은

> **리스크를 얼마나 빠르게 발견하고 대응할 수 있는가**

였습니다.

단순히 Error를 발생시키는 것이 아니라

- Error Class
- Error Code
- Error Level

을 통해 캡슐화하여 관리하도록 설계하였습니다.

## Error Level

| Level   | 의미     | 예시                        | 처리 방식      |
| ------- | -------- | --------------------------- | -------------- |
| Level 1 | INFO     | 증빙 완료, 승인 완료        | 일반 알림      |
| Level 2 | WARNING  | 증빙 누락, 부가세 위험      | 회계 검토 요청 |
| Level 3 | CRITICAL | VIN 압류, 도난 의심, 역마진 | 즉시 대응      |

---

## Critical 처리

다음과 같은 상황은

- VIN 압류
- VIN 도난 의심
- 역마진 발생
- 선적 불가 차량

모두 **Critical** 로 분류됩니다.

향후 실제 서비스에서는

- Slack
- Discord
- Microsoft Teams

등 메신저와 연동하여 즉시 알림을 보낼 수 있도록 고려하였습니다.

# 🚢 선적 프로세스 개선

## 기존 프로세스

```text
전체 재고

↓

VIN 대기

↓

선적 보류

↓

조건부 선적

↓

선적 완료
```

실제 업무에서는

중간마다 선적하지 못하는 이유가 존재합니다.

예를 들어

- 현금영수증 미발행
- 증빙 누락
- 부가세 손실
- VIN 검증 실패

등 다양한 리스크가 존재합니다.

---

## 개선된 프로세스

```text
재고

↓

VIN 검증

↓

세무 검토

↓

증빙 검토

↓

승인 대기

↓

선적 가능

↓

선적 완료
```

---

## 회계팀 직권 보류

회계팀은 언제든

**직권 선적 보류**

를 실행할 수 있도록 구현하였습니다.

직권 보류 시

- 차량은 승인대기 상태로 변경
- 영업팀은 임의 선적 불가

즉,

회계팀의 통제권을 시스템에서 보장하도록 설계하였습니다.

# 📄 증빙 관리

증빙 누락이 자주 발생한다고 판단하였습니다.

따라서

- 증빙이 부족한 차량만 조회
- 여러 건 선택 후 일괄 처리

가 가능하도록 구현하였습니다.

# 💡 프로젝트에서 중요하게 생각한 점

이 프로젝트는

단순한 차량 관리 프로그램이 아니라

> **회계부서가 의사결정 권한을 가지며 직원들이 이용할 수 있는 플랫폼**

을 목표로 설계하였습니다.

차량을 단순히 등록하는 것이 아니라

**차량의 위험도를 자동으로 판단하고**

회계팀이 빠르게 대응할 수 있도록 하는 것입니다.
